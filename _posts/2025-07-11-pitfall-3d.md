## About the Project

My second project at BUas, an endless runner built in a custom 3D engine targeting the Raspberry Pi. Despite the Pitfall name given by the school, the result ended up closer to Temple Run than the classic Atari game.

**Team:** solo
**Duration** 8 weeks
**Engine:** Custom engine  
**Play it:** (no build available)

---

## Overview

Building on the previous project, this time I was working in 3D with OpenGL on a platform other than PC. The engine was custom built and had to run on the Raspberry Pi, which added hardware constraints on top of everything else.

On top of the renderer I built a tile-based level system that spawns sections ahead of the player with randomised obstacles and collectables, integrated Bullet for 3D physics and collision detection, implemented skeletal animations, and added an ImGui panel for adjusting the character mesh scale and rotation at runtime.

The game is an endless runner where the player moves through a procedurally generated level, collecting pickups and avoiding fence obstacles. One of the more interesting features was a gravity system with three modes, letting the player run along different orientations. An ImGui panel allowed adjusting the character mesh scale and rotation at runtime, which was useful for testing.

## What I Learned

- **OpenGL** — my first hands-on experience with a graphics API
- **Bullet physics** — integrating a third party physics library for 3D collision detection
- **Skeletal animation** — getting animated characters working inside a custom engine
- **Procedural generation** — building a system to generate the level layout at runtime
- **ImGui** — building a live editor tool for in-game tweaking
- **Raspberry Pi / platform targeting** — working within the constraints of non-PC hardware
- **3D fundamentals** — cameras, transforms, and thinking in three dimensions for the first time
